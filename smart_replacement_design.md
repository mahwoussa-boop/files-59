# تصميم المحرر الذكي للاستبدال والتحرير

النسخة القادمة يجب أن تعالج مشكلتين أساسيتين: عدم ظهور النصوص القابلة للتحرير في بعض الصفحات، وصعوبة استبدال كلمة معيّنة مباشرة دون المرور بخطوات يدوية كثيرة.

## المسارات الجديدة

| المسار | الهدف |
|---|---|
| **استخراج نص أصلي** | استخدام طبقة النص الأصلية من ملف PDF عندما تكون متاحة |
| **OCR احتياطي** | استخراج الكلمات من الصفحة المصورة عندما لا توجد طبقة نصية قابلة للتحرير |
| **استبدال ذكي** | البحث عن كلمة أو عبارة وتحديد أفضل تطابق ثم استبداله مباشرة |
| **مساعدة AI اختيارية** | استخدام نموذج ذكي لترجيح أفضل عنصر عند فشل المطابقة الحرفية |

## سلوك الواجهة

سيُضاف قسم جديد باسم **الاستبدال الذكي** يسمح للمستخدم بإدخال:

1. النص أو الكلمة المراد استبدالها.
2. النص البديل.
3. تفعيل OCR الاحتياطي عند عدم وجود نصوص أصلية.
4. تفعيل المساعدة الذكية عند عدم العثور على تطابق مباشر.

عند التنفيذ سيحاول التطبيق بالترتيب التالي:

1. المطابقة الحرفية المباشرة.
2. المطابقة المطبّعة بعد تنظيف النص.
3. البحث الجزئي داخل النصوص.
4. إذا فشل كل ما سبق وكان الذكاء الاصطناعي مفعّلًا، يرسل التطبيق قائمة النصوص إلى النموذج الذكي ليختار أقرب عنصر.

## التعديلات البرمجية

| الملف | التعديل |
|---|---|
| `pdf_editor.py` | دعم OCR، وتوحيد تمثيل العناصر النصية، وإضافة دوال البحث الذكي والاستبدال |
| `app.py` | إضافة قسم الاستبدال الذكي وخيارات OCR وAI |
| `utils.py` | إضافة تطبيع النصوص العربية والإنجليزية للمطابقة الذكية |
| `requirements.txt` | إضافة مكتبات OCR وعميل الذكاء الاصطناعي |
| `ai_helper.py` | ملف جديد للمساعدة الاختيارية في اختيار أفضل تطابق |

## الهدف النهائي

أن يستطيع المستخدم كتابة كلمة مثل "الاسم" ثم يكتب الكلمة البديلة، فيقوم التطبيق بمحاولة ذكية لتحديد موقعها، حذفها بصريًا، وإدراج النص الجديد في **نفس الموضع واللون والحجم والخط التقريبي**، حتى لو كانت الصفحة صورية ويحتاج الأمر إلى OCR.
